# Code Review / Debugging Exercise

Review `buggy-robot.ts` and identify issues by category:

1. Correctness bugs
2. Design and maintainability problems
3. Testing gaps
4. Runtime or API safety risks

Then propose a fixed version and a minimal test plan.

## Optional timed format
- 7 minutes: read and annotate issues
- 8 minutes: discuss fixes and tradeoffs
- 5 minutes: write one critical test first
